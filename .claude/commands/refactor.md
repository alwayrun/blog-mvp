# 重构指定模块

## 用法
/refactor <模块名或文件路径>

## 执行步骤

### 1. 分析现状
读取目标文件，列出：
- 当前存在的问题（重复代码、类型不安全、职责不清等）
- 重构目标和预期收益
- 影响范围（哪些文件依赖这个模块）

**暂停，等用户确认重构方向后再继续。**

### 2. 制定计划
列出具体改动步骤，确保每步都是可独立验证的。

### 3. 执行重构
- 保持对外接口不变（函数签名、返回类型）
- 一次只改一个文件，改完运行类型检查
- 不在重构中混入新功能

### 4. 每步验证
```bash
npx tsc --noEmit
```

每修改一个文件后运行，保持零错误。

### 5. 最终验证

```bash
npm run build
```

### 6. 对比说明

输出重构前后的对比：
- 代码行数变化
- 解决了哪些具体问题
- 是否有行为变化（理论上不应该有）
